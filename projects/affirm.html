Designing internal software for financial technology

Affirm's support team struggled to manage customer accounts due to inefficient workflows, often causing servicing errors and customer frustration. Alongside a product team, I designed new internal tooling, called Agent Portal, to address their problems and create a better experience.

Problems

  • Different accounts belonging to the same customer had to be managed separately.
  • This separation of accounts also led to data discrepancies, such as multiple addresses listed for the same customer.

Solutions

  • Design internal tooling that unifies all of a customerâ€™s accounts into a single page.
  • Allow agents to easily handle data discrepancies.

Welcome and Search

When searching, support agents can now see all associated accounts under a single name.

Main Pages

In the Overview, agents can view info and which accounts the info is associated with. The Manage Person page provides a centralized page for most account-level interactions.

Loans

Viewing a loan application or notification reveals a side-panel with detailed info regarding that item.

Editing Contact Info

When an agent updates any information in the new interface, the update propagates to the customer's other accounts.

Merging Conflicting Data

Affirm databases were littered with data discrepancies in customer accounts. Support agents can now resolve these conflicts in a few clicks.

Downloading Documents

Previously, agents had to download all of a customer's documents into a single .zip file. The new interface allows agents to download just what they need.
